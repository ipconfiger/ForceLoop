📋 调研结论速览

核心结论：两边底层都是「YAML frontmatter + markdown body」结构，可以共用，但需要一层薄薄的转换层（codegen），不存在 100% 字段重合的零摩擦方案。

最大公约数：name · description · model · argument-hint（仅 Claude）· 工具白名单（语法不同）

最低成本实现：写一份 ForceLoop 自己的 schema + 一个 forceloop command compile codegen，把同一份 .md 转成两边原生格式。

4. 关键差异（影响通用格式设计）

维度Claude CodeOpenCode
参数占位符 $ARGUMENTS, $1/$2 (旧) 仅 $ARGUMENTS
工具过滤语法 逗号列表 "Bash, Read, Edit" + 可能的 Bash(git:*) 前缀 permissions: { bash: "allow", edit: "deny" } 风格
过滤方向 默认全开 → allowed-tools 白名单; agent 用 disallowedTools 黑名单 v1 布尔 map; v2 用 permissions (allow/deny/prompt/ask)
子 agent 提示词结构 XML <Agent_Prompt> 块 (Role/Constraints/Criteria) 单字符串 system: ...
MCP 工具引用 mcp__server__tool 前缀 mcp__server__tool 同样前缀 (语义/实现不同)
Shell escape 任务代理手动调 命令体内置 ! `...`
文件引用 隐式 (用工具读) 内置 @file/path 占位符
Skill 触发 靠 description 模糊匹配自动 slash: true 显式 / 隐式自动

5. 通用格式建议

5.1 最小可用通用子集（两边都支持）

---
name: <id>                          # ✅ 都有
description: <text>                 # ✅ 都有
model: <model-id>                   # ✅ 都有 (仅两边都允许的位置)
---

# body (markdown)

Arguments: $ARGUMENTS

加上可选：

---
allowed-tools: ["Bash", "Read"]          # Claude 原生
# OR
permissions: { bash: "allow", edit: "deny" }   # OpenCode 原生
---
建议：使用 YAML list 风格（["Bash", "Read"]），而非 Claude 的逗号字符串——OpenCode 完全接受 list，Claude 也接受 list（逗号字符串是简写）。

6. ForceLoop 落地建议

如果想实现「一份定义，两边跑」：

  1. 写一份 ForceLoop 自己的 command schema（name, description, model, tools, argument-hint, agent, prompt），存到 .forceloop/commands/<name>.md
  2. 加一层 codegen 把 ForceLoop schema 转成两边原生格式：
    • → Claude: description, allowed-tools: [list], body 直接用 prompt
    • → OpenCode: description, model, agent, body 直接用 prompt（v2 暂不支持 tools 在 command 上，转换成把 permissions 透传到所委派的 agent）
  3. 不通用字段（disable-model-invocation / subtask / slash）作为「目标平台特定覆盖」放在 platforms: { claude: {...}, opencode: {...} } 子表里

6.1 ForceLoop 通用 command 草案

---
name: code-review
description: "Review changed files across dimensions with severity-rated feedback"
model: opus
tools: [Read, Grep, Bash]              # 通用工具白名单 (list 形式)
argument-hint: "[files...]"
agent: code-reviewer                    # 委派给哪个子 agent
---

You are Code Reviewer. Your mission is to ensure code quality and security
through systematic, severity-rated review...

→ 转 Claude

description: "Review changed files across dimensions with severity-rated feedback"
allowed-tools: [Read, Grep, Bash]
model: claude-opus-4-6

body 直接 = prompt。

→ 转 OpenCode

description: "Review changed files across dimensions with severity-rated feedback"
model: opus
agent: code-reviewer

body 直接 = prompt（如果 command 没有 tools 字段，权限由所委派 agent.permissions 提供）。

7. 不确定 / 待验证事项

⚠️ Unverified 清单

  • OpenCode 全局路径 ~/.config/opencode/... 未本地验证（子 agent 报告标记为 unverified）
  • OpenCode subtask 字段语义（同步? 异步? 后台?）— 源码声明存在，consumer 未追踪
  • OpenCode MCP 集成：config/mcp.ts 文件在远端 git 拉取超时，未拿到 schema
  • Claude Code model 字段在 command 中是否生效（本地样本里只在 agent 里用 model）
  • Claude Code allowed-tools 的 Bash(git:*) 前缀语法（子 agent 报告里标记为 unverified）
  • OpenCode $ARGUMENTS 插值规则（$1/$2 位置参数既未确认支持也未排除）
